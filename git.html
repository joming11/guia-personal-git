Flujo Basico de Git

Estados de Git

  1. Working directory(modified)
  2. staging area(staged)
  3. commit (commited)
  4. Remote

el Working directory es el area de trabajo, la carpeta en si, donde inciamos el repositorio es donde se modificaran, eliminaran o agregaran archivos, por eso es el estado que esta en constante modificacion.

el Staging area, es el area donde estan los archivos que van a ser confirmados y enviados a HEAD la cabecera del repositorio, en este estado aun pueden ser devueltos al working directory para una nueva modificacion o para ser confirmados y guardarlos en el repositorio.

HEAD, es el estado donde estan confirmados los archivos

El estado remoto es cuando ya las carpetas se encuentran en un sistema como github, gitlab, bitbucket

Comandos para trabajar en el flujo basico de Git

git add archivo -> para agregar un elemento al staging

git add . -> para agregar mas de un elemento al staging

git commit para confirmar un archivo y enviarlo a la cabecera del repositorio al no tener la bandera -m se abrira el editor para definir un mensaje del commit

git commit -m para describir en un mensaje de la confirmacion

Github

para enlazar el repositorio local con un repositorio en github

se crea un nuevo repositorio en github, nombre y descripcion se agrega un readme y un .gitignore si no se han creado previamente y se agrega una licencia

Al crear un repositorio en github dara las indicaciones para crear el repositorio local desde la terminal. crear la carpeta, el readme, git init, git add, git commit, git branch -M main git remote add origin https://url git push -u origin main actualmente, la convencion es usar main como nombre de rama principal, por eso se hace el push -u origin desde main

si ya se ha creado el repositorio local, github dara las opciones para enlazarlo git remote add origin https://url para enlazar los repositorios git branch -M main git push -u origin main para subir los archivos

una vez que los repositorios, local y remoto esten enlazados y se hagan nuevos cambios para subirlos a github se utlizaria solo git push ya no seria necesario colocar -u origin

Si se realizara un cambio desde github, para bajarlo al repositorio local se usa git pull

Ramas en git (branch)

todo el desarrollo de un repositorio ocurre en una rama, por lo general es o era la rama master, por tema de inclusion, ahora la rama principal por convencion se llama main para pasar el desarrollo de la rama master a la rama main, hacemos: git branch -m master main -se crea la rama local y se le pasa el historial de la rama master git branch -m master main

Se hace un push de la nueva rama principal al repositorio remoto

se cambia el HEAD actual a la rama main git symbolic-ref refs/remotes/origin/HEAD refs/remotes/orign/main
