Flujo Basico de Git

Estados de Git

  1. Working directory(modified)
  2. staging area(staged)
  3. commit (commited)
  4. Remote

el Working directory es el area de trabajo, la carpeta en si, donde inciamos el repositorio es donde se modificaran, eliminaran o agregaran archivos, por eso es el estado que esta en constante modificacion.

el Staging area, es el area donde estan los archivos que van a ser confirmados y enviados a HEAD la cabecera del repositorio, en este estado aun pueden ser devueltos al working directory para una nueva modificacion o para ser confirmados y guardarlos en el repositorio.

HEAD, es el estado donde estan confirmados los archivos

El estado remoto es cuando ya las carpetas se encuentran en un sistema como github, gitlab, bitbucket

Comandos para trabajar en el flujo basico de Git

git add archivo -> para agregar un elemento al staging

git add . -> para agregar mas de un elemento al staging

git commit para confirmar un archivo y enviarlo a la cabecera del repositorio al no tener la bandera -m se abrira el editor para definir un mensaje del commit

git commit -m para describir en un mensaje de la confirmacion
